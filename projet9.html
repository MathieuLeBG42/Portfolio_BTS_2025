Projet : Automatisation de l’installation de Windows avec AutoUnattend.xml

Dans le cadre de ma formation BTS SIO SISR, j’ai réalisé un projet visant à automatiser l’installation de Windows à l’aide d’un fichier AutoUnattend.xml. L’objectif : faciliter le déploiement de systèmes Windows en environnement professionnel, réduire les interventions humaines et uniformiser les installations.

Interface du site de Schneegans

Pourquoi utiliser AutoUnattend.xml ?

Le fichier XML permet de personnaliser complètement l’installation de Windows : création automatique d’utilisateurs, désactivation de fonctionnalités et logiciels Windows, préinstallation de logiciels, personnalisation graphique, et configuration réseau automatisée. Il réduit fortement le temps et les erreurs humaines lors des déploiements.

Quelques fonctionnalités désactivables depuis le site de Schneegans

Étapes de mise en œuvre

1. Analyse du besoin

  • Déploiement rapide et homogène de Windows
  • Suppression des étapes manuelles et des choix par défaut lors de l’installation
    • Windows installé avec le fichier AutoUnattend.xml non modifié

    2. Création du fichier XML

  • Utilisation du site Unattend Generator pour créer un fichier AutoUnattend.xml
  • Personnalisation des comptes utilisateurs, options de mise à jour, et programmes à installer
    • Extrait d'un fichier AutoUnattend.xml

    3. Fonctions automatisées incluses

  • Création automatique de comptes et définition des mots de passe
  • Désactivation possible de Defender, Windows Update, applications inutiles
  • Exécution automatique de scripts PowerShell
  • Installation d'applications à l'aide de commandes winget
    • Activation des scripts à l'installation de Windows

    4. Intégration du fichier dans l'ISO Windows

  • Intégration du fichier AutoUnattend.xml dans l'ISO à l'aide du script de WIMutil avec la commande :
    • irm "https://github.com/memstechtips/WIMUtil/raw/main/src/WIMUtil.ps1" | iex
  • L'outil décompile les fichiers d'un ISO Windows cible et intègre automatiquement le XML avant de le recompiler
  • Une fois le XML intégré, ce dernier sera automatiquement pris en compte lors du démarrage sur l'ISO
    • Interface de l'outil WIMUtil

    5. Résultats obtenus

  • Installation complète sans interaction utilisateur
  • Personnalisation appliquée dès le premier démarrage
  • Gain de temps significatif et processus réplicable en entreprise
    • Windows configuré par XML

    6. Avantages et limites

  • Automatisation poussée : + rapide, + fiable, + cohérent
  • Peut être intégré dans un processus PXE ou USB boot
  • Nécessite de nombreux tests afin d'obtenir un résultat poussé
